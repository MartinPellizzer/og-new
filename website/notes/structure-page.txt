## **Step-by-Step Plan for the Header**

### **Step 1 — Define the HTML Structure**

1. **Use `<header>` semantic element** as the wrapper.
2. Inside `<header>`, include:

   * A **logo container** (`<div class="logo">`), preferably `<a href="index.html"><img src="logo.png" alt="Ozonogroup"></a>`)
   * A **navigation container** (`<nav>`), containing an unordered list of links.
3. Navigation links for now:

   * Applicazioni
   * Prodotti
   * Servizi
   * Guida all’Ozono
   * Contatti / Chi Siamo
   * Optional: “Problemi” or “Blog” if needed in the future

**Desktop layout:** Logo left, nav links right, horizontally aligned.

**Mobile layout:**

* Collapse navigation into a **hamburger menu icon** (toggle visibility).
* When clicked, nav links expand vertically below the header.
* Hamburger should be accessible (`<button>` with aria-label).

---

### **Step 2 — Create the CSS Layout**

**Desktop:**

* Header: sticky, top: 0, white background, subtle shadow.
* Flex container: `display: flex; align-items: center; justify-content: space-between`.
* Padding: consistent with your style guide (e.g., `var(--space-md) var(--space-lg)`).
* Logo: fixed height (e.g., 50–60px), auto width.
* Nav links: horizontal list, spacing `gap: var(--space-lg)`.
* Link styles: follow current color / hover styles.

**Mobile:**

* Use media query `max-width: 768px`.
* Hide nav links by default, show **hamburger menu button**.
* On toggle, display links vertically with padding.
* Smooth transition for expand/collapse.
* Hamburger icon can be CSS-based or inline SVG.

---

### **Step 3 — Accessibility Considerations**

* `<nav aria-label="Main Navigation">` for semantic navigation.
* Hamburger button:

  * `<button aria-label="Toggle navigation menu" aria-expanded="false">`
  * Update `aria-expanded` when menu opens/closes.
* Focus states: use your existing `:focus` style for keyboard navigation.
* Alt text for logo image (`alt="Ozonogroup"`).

---

### **Step 4 — Optional Enhancements**

* Sticky header with **subtle shadow**: helps differentiate from content.
* Dropdown menus for future expansion (e.g., “Prodotti” → product categories).
* Slight hover animation on links (e.g., color change, underline slide).
* Mobile menu sliding from top or side (optional later).
* Ensure **high contrast** for trust & professional feel.

---

### **Step 5 — Implementation Plan**

1. **HTML:**

   * Create `<header>` with logo `<div>` + `<nav>`
   * Add `<ul>` for links inside nav

2. **CSS:**

   * Flex layout for desktop

   * Link styling (color, hover, focus)
   * Mobile media query for hamburger menu
   * Transition for mobile menu expand/collapse
3. **Optional JS (later):**

   * Toggle mobile menu on button click
   * Update `aria-expanded` attribute
